# 前端最小可用回归清单（P0+）

> 用途：PR 阶段最小回归基线。每个核心页面保持 2~3 条可执行用例，新增功能需补齐对应条目。

## 1. 登录页 `/#/login`

- 输入正确用户名/密码后跳转到 `/#/admin/dashboard`。
- token 过期时访问受保护页面会被路由守卫重定向到登录页。

## 2. 仪表盘 `/#/admin/dashboard`

- 页面可渲染核心模块（系统信息/聊天信息）。
- dashboard 数据接口返回空数据时页面不崩溃。

## 3. 模块统计 `/#/admin/module-metrics`

- 卡片列表可加载并支持关键词筛选。
- 可从总览进入模块详情并返回总览。
- 筛选参数变化后请求参数正确。

## 4. 插件策略 `/#/admin/plugin-policy`

- 策略矩阵可加载并展示分组/插件。
- 切换插件开关会发送更新请求，`gid/plugin_name/enabled` 正确。

## 5. 记忆管理 `/#/admin/memory-admin`

- 页面初始化会加载会话列表、节点、关联数据。
- 切换会话后会重新拉取图谱数据并更新“当前视图”。
- 节点/关联统计展示与接口返回一致。

## 6. 数据库管理 `/#/admin/db-admin`

- SQL 数据源下可选择表并触发查询。
- SQL 查询结果可在结果区正常展示。
- 切换到 Neo4j 数据源后可执行 Cypher 并返回结果。

## 执行策略

- PR 必跑：`npm run test:e2e:smoke`
- Nightly 全量：`npm run test:e2e`
- 新增核心页面时：同步新增本清单条目 + 至少 1 条 smoke 用例
